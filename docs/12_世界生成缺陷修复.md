# 世界生成链路两处缺陷修复

> 时间：2026-07-09｜范围：`index.html` 真实代码（非重写逻辑）
> 背景：docs/11 二轮测试末尾列出的两项后续优化，本质是会让"玩家自建世界"体验受损的代码缺陷，用户确认"先修复代码缺陷"。

---

## 缺陷 1：`callWorldGenerationLLM` 缺 `max_tokens`（世界 JSON 易被截断）

**位置**：`callWorldGenerationLLM`（约 2510 行）的 fetch body。

**问题**：原请求体只有 `model / messages / temperature / response_format`，依赖服务端默认 `max_tokens`。AI 生成一个大世界需要输出 `lore_kb`（8–12 条 × 100–300 字）+ `initial_state` + `system_prompt` + `opening_narrative` 等结构化 JSON，极易超过服务端默认上限被截断 → 截断的 JSON 经 `parseResponse`/`tryRepairJSON` 修不好就直接"生成失败"，用户看到报错。

**修复**：补 `max_tokens: 8192`（与全局对话调用统一，足够容纳典型大世界，且不会触发模型输出上限）。

```js
body: JSON.stringify({
    model,
    messages: [{ role: "system", content: prompt }],
    temperature: 0.7,
    max_tokens: 8192,                 // ← 新增
    response_format: { type: "json_object" }
})
```

---

## 缺陷 2：AI 生成世界的 `lore_kb` 永远没有 embedding（向量检索对其失效）

**位置**：`embeddingRetrieve`（约 3597 行）、`generateWorld`（约 2415 行）；`ensureLoreEmbeddings`/`computeEmbedding` 此前不存在。

**问题**：`embeddingRetrieve` 开头 `if (!kb.snippets[0].embedding) return []`——只要首片段无向量就整体返回空。预生成世界（`data/lore_kb_with_embeddings.json`）带着 384 维预生成向量，所以 demo 世界正常；但 **AI 实时生成的世界 `lore_kb` 从没算过 embedding**，于是这些世界的向量检索永久失效，只剩关键词检索，RAG 召回质量下降。

**修复**：补齐"生成后补算 + 兜底补算"两段能力。

1. 新增 `computeEmbedding(text)`：复用已加载的 `transformers.pipeline("feature-extraction", "Xenova/all-MiniLM-L6-v2")`，`pooling:"mean"` + `normalize:true`，返回 `Array.from(out.data)`。
2. 新增 `ensureLoreEmbeddings(kb)`：为 `lore_kb.snippets` 中缺 `embedding` 的片段补算并写回（随世界持久化）；`transformers` 不可用或模型加载失败时静默降级，不抛错。
3. `generateWorld()` 在拿到 AI 生成结果后、组装 `world` 前：`await ensureLoreEmbeddings(generated.lore_kb)`。
4. `embeddingRetrieve()` 顶部加兜底：`if (!kb.snippets[0].embedding) await ensureLoreEmbeddings(kb)`（覆盖老存档 / 生成时跳过的世界），随后仍无向量才 `return []`。

```js
async function computeEmbedding(text) {
    if (typeof transformers === "undefined") throw new Error("transformers 不可用");
    if (!embeddingModel) {
        embeddingModel = await transformers.pipeline("feature-extraction", "Xenova/all-MiniLM-L6-v2");
    }
    const out = await embeddingModel(text, { pooling: "mean", normalize: true });
    return Array.from(out.data);
}

async function ensureLoreEmbeddings(kb) {
    if (!kb || !kb.snippets || !kb.snippets.length) return;
    if (kb.snippets[0].embedding) return;
    if (typeof transformers === "undefined") return;
    for (const s of kb.snippets) {
        if (s.embedding) continue;
        const text = [s.category, s.title, s.content, (s.keywords || []).join(" ")].filter(Boolean).join(" ");
        try { s.embedding = await computeEmbedding(text); }
        catch (e) { console.warn("知识库片段向量计算失败:", e.message); return; }
    }
}
```

**降级保障**：若浏览器环境无法加载 transformers.js（离线 / CDN 受限），`ensureLoreEmbeddings` 与 `embeddingRetrieve` 均不抛错——前者静默跳过、后者降级返回 `[]`，游戏继续以纯关键词检索运行，**绝不崩溃**。

## 验证（vm 加载真实代码，离线 10/10 全过）

| 项 | 断言 | 结果 |
|---|---|---|
| A. `callWorldGenerationLLM` | fetch body 含 `max_tokens=8192` 且 `parseResponse` 正常解析出 `lore_kb`（3 条） | PASS |
| B. `ensureLoreEmbeddings` | 为每个片段补算向量（维度 16 桩） | PASS |
| B. `embeddingRetrieve` | 返回命中结果，top 余弦 `0.5477` > 0，最相关片段 = 林黛玉 | PASS |
| C. 降级 | `transformers` 缺失时 `ensureLoreEmbeddings` 静默跳过、`embeddingRetrieve` 返回 `[]` 且不崩溃 | PASS |

> 说明：验证用桩 `fakeVec` 初版因文件源码里正则 `\\s` 被解析为"字面反斜杠+s"（不匹配空白）导致余弦恒为 0，属测试桩转义 bug，修正为 `\s` 后全过；非 index.html 代码缺陷。

## 影响评估

- **老存档**：无影响。`migrateGameState` 不涉及向量；老世界加载时若缺 embedding，`embeddingRetrieve` 兜底补算（需 transformers）。
- **预生成向量缓存**：无影响。`data/lore_kb_with_embeddings.json` 未被改动，demo 世界仍走既有 384 维预生成向量。
- **成本**：仅在"创建世界"时一次性补算向量；首次进入游戏时若尚未补算也会触发一次，之后随世界持久化，不再重复计算。

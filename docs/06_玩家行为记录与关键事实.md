# AI 文字游戏 · 玩家行为记录与关键事实

玩家行为记录（Behavior Records）是动态 RAG 的核心，用于让 AI 记住玩家做过的事，保持多轮叙事一致性。

## 1. 什么是关键事实

关键事实是从每次玩家行动中提炼出的、对后续剧情有影响的简短陈述：

- 玩家获得了什么物品
- 玩家到达了什么地点
- 玩家与某 NPC 的关系变化
- 玩家完成了什么事件
- 玩家触发了什么状态效果
- 玩家死亡原因

## 2. 存储格式

```json
{
  "behavior_records": [
    {
      "id": "b123",
      "text": "玩家在小镇入口从药铺掌柜处获得了草药 x1",
      "createdAt": "2026-06-29T12:00:00Z"
    }
  ]
}
```

## 3. 提取方式

### 3.1 LLM 自动提取

在 System Prompt 中要求 LLM 在返回 JSON 时附带 `key_facts` 字段：

```json
{
  "key_facts": [
    "玩家获得了草药",
    "玩家与药铺掌柜的关系从 0 提升到 2",
    "玩家完成了事件：初次购药"
  ]
}
```

优点：语义理解强，能提炼隐含事实。

### 3.2 规则提取

从 `state_changes` 中直接提取：

```javascript
function summarizeFactsFromChanges(input, narrative, changes) {
    const facts = [];
    if (changes.inventory) {
        for (const op of changes.inventory) {
            if (op.op === "add") facts.push(`玩家获得了 ${op.name} x${op.count}`);
        }
    }
    if (changes.relationships) {
        for (const [k, v] of Object.entries(changes.relationships)) {
            if (v > 0) facts.push(`玩家与 ${k} 的关系有所提升`);
        }
    }
    if (changes.completed_events) {
        for (const e of changes.completed_events) facts.push(`玩家完成了事件：${e}`);
    }
    if (changes.current_location) facts.push(`玩家到达了 ${changes.current_location}`);
    return facts.slice(0, 5);
}
```

优点：稳定、不依赖 LLM 额外输出。

## 4. 使用方式

每次玩家输入后，检索行为记录中与当前输入相关的事实，注入 System Prompt：

```markdown
# 玩家行为记录（关键事实）

- 玩家在小镇入口获得了草药 x1
- 玩家与老道长的关系有所提升
- 玩家完成了事件：找到落脚之处
```

AI 看到这些事实后，会在叙事中自然呼应，避免失忆。

## 5. 检索策略

对行为记录做关键词匹配即可：

```javascript
function retrieveBehaviorRecords(input, topK) {
    const terms = input.toLowerCase().split(/\s+/).filter(t => t.length >= 2);
    return behavior_records
        .map(b => ({ ...b, score: terms.filter(t => b.text.includes(t)).length }))
        .filter(x => x.score > 0)
        .sort((a, b) => b.score - a.score)
        .slice(0, topK);
}
```

如果行为记录过多，可只保留最近的 100 条。

## 6. 去重与更新

避免重复记录完全相同的事实：

```javascript
if (!behavior_records.some(b => b.text === text)) {
    behavior_records.push({ id, text, createdAt });
}
```

对于关系变化，可保留最新的一条而不是累加重复。

## 7. 定期摘要

当行为记录过多时，可让 LLM 定期摘要：

> 请将以下玩家行为记录摘要成 5-10 条关键事实，保留最重要的信息。

摘要后替换旧记录，减少 prompt 长度。

## 8. 注意事项

- 关键事实要简洁，每条控制在 30 字以内。
- 不要记录无意义的随机行动。
- 涉及剧情转折、NPC 态度变化、重大获得/失去的事实要优先保留。
- 玩家死亡后，相关行为记录仍可保留，用于复盘。

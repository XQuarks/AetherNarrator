# AI 文字游戏 · 知识库设计

知识库（Lore Knowledge Base）是 RAG 检索的核心，用于约束 AI 不瞎编、不跨界。

## 1. 知识库结构

```json
{
  "ip": "剑来",
  "snippets": [
    {
      "id": "x1",
      "category": "规则",
      "title": "修行境界",
      "content": "凡人、练气、筑基、金丹、元婴……境界不可跳跃，需按部就班修行。",
      "keywords": ["境界", "修行", "突破"]
    }
  ]
}
```

## 2. 片段分类

| category | 用途 | 示例 |
|----------|------|------|
| 规则 | 世界运行的硬规则 | 境界顺序、魔法限制、因果律 |
| 地点 | 场景设定与限制 | 小镇、学院大厅、禁地 |
| 人物 | NPC 性格与态度 | 老道长、分院帽、魔药教授 |
| 事件 | 关键剧情节点 | 分院仪式、宗门大比 |
| 物品 | 重要道具 | 魔杖、丹药、法宝 |
| 势力 | 组织与派系 | 四大学院、三大圣地 |
| 世界观 | 整体氛围与风格 | 黑暗森林、热血升级 |

## 3. 片段长度

- 每条片段控制在 **100-300 字**。
- 太长会导致检索时噪声多，太短会丢失上下文。
- 复杂规则拆成多条，每条讲清一个点。

## 4. 关键词设计

- 每条片段必须包含 `keywords` 数组，3-8 个词。
- 关键词用于关键词检索和粗排。
- 应覆盖：名称、别名、相关地点、相关人物、核心概念。

## 5. 向量嵌入

为支持语义检索，每条片段需要生成 embedding：

```json
{
  "id": "x1",
  "category": "规则",
  "title": "修行境界",
  "content": "...",
  "keywords": ["境界", "修行"],
  "embedding": [0.12, -0.05, ...]
}
```

生成方式：

- 模型：`sentence-transformers/all-MiniLM-L6-v2` 或 `bge-m3`。
- 文本拼接：`[category] title\ncontent`。
- 工具：见 `tools/generate_embeddings.py`。

## 6. 检索策略

运行时，根据玩家输入同时做：

1. **关键词检索**：按关键词匹配 snippets，快速召回。
2. **向量检索**：按语义相似度召回。
3. **行为记录检索**：检索玩家关键事实。

最终融合排序，取 Top-K（建议 6-10 条）注入 System Prompt。

## 7. 必须注入的硬规则

无论检索结果如何，以下片段应始终注入：

- 力量体系核心规则（境界顺序、魔法限制等）。
- 玩家当前地点的相关规则。
- 当前涉及 NPC 的性格与态度。
- 世界观禁忌（不可做之事）。

## 8. 动态知识库

创建世界时，AI 应根据世界观描述自动生成 lore_kb。生成后允许玩家后续手动补充或修正。

## 9. 维护建议

- 定期审查 AI 是否频繁编造不属于知识库的内容。
- 发现编造后，将正确信息补充进知识库。
- 对玩家行为记录（关键事实）定期归档，避免无限增长。

# AI 文字游戏 · 示例：修仙世界

本示例展示如何为一个修仙世界观生成完整的游戏配置。

## 1. 用户输入

- **世界名称**：剑来
- **类型**：ip
- **世界观描述**：大千世界，万族林立。修行者通过吸纳天地灵气提升境界，境界分为凡人、练气、筑基、金丹、元婴等。世界中有宗门、散修、妖兽、洞天福地。修行者讲究因果、机缘、心性。
- **主角设定**：小镇少年，性格谨慎坚韧，因一次意外获得半部残卷。

## 5. System Prompt 节选

```markdown
你是《剑来》背景文字游戏的主持人。规则：

1. 境界顺序为 凡人→练气→筑基→金丹→元婴，不可跳跃。
2. 老道长不会无条件帮助玩家，需要关系或代价。
3. 玩家持有残破玉简，可据此展开功法线索。
4. 属性、关系、技能全部用文字描述，只在有显著变化时更新。
5. 每次输出 2-4 个选项，同步输出 JSON 状态变更。
6. 玩家死亡时设 is_alive=false。
```

## 6. 角色面板展示

- **属性页签**：胆识、洞察、耐心、气运、心志（均为文字描述）
- **进度页签**：修行路线 / 境界 / 进度
- **技能页签**：功法/技艺（文字描述）

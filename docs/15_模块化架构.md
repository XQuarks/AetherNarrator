# 模块化架构（M0–M5）

将原单文件 `app.js`（约 4300 行 / 151 函数 / 36 个全局状态）按领域拆分为 11 个 ES Module，浏览器原生 `import` 加载，零构建、GitHub Pages 直接部署。

## 1. 为什么选 ES Module 多文件

目标是"用户点开 GitHub Pages 网址直接游玩"。ESM 多文件是该目标下的最优解：

- GitHub Pages 是标准 HTTP 服务器，原生支持 `<script type="module">` 与 `import`，`.js` 返回正确的 `text/javascript` MIME。
- 全部使用相对路径，`deploy.yml` 零构建（`upload-pages-artifact path: .`）即可上线。
- 与现有 CSP / SRI 兼容，无需打包器。

**红线**：所有 `import` 必须用 `./xxx.js` 相对路径。禁用前导斜杠的绝对路径（`/src/x.js`），否则部署到 `user.github.io/<repo>/` 子路径会 404。跨域 DeepSeek 直连受浏览器 CORS 限制，公开部署需自备代理。

## 2. 核心架构决策：全局状态收进 `S` 容器

36 个可变全局状态**全部收进 `store.js` 的单一容器对象 `S`**，跨模块读写统一 `S.xxx`：

```js
// store.js
export const S = { gameState: null, currentWorld: null, /* …36 项 */ };
```

放弃 `export let x` 的 live binding 方案，原因：

- `export let` 的绑定在导入侧是**只读**的，跨模块重赋值（`x = ...`）会 TypeError。
- 容器对象只需机械地给状态引用加 `S.` 前缀，赋值 `S.x = ...` 天然可写，且与循环依赖无关。
- 模块私有状态也一并进 `S`，消除"私有/核心"误判风险，简化抽取。

## 4. 拆分方法：真 JS 解析器（acorn）

放弃正则/括号计数切分（模板字符串、正则字面量、转义会导致边界静默错误）。改用 **acorn 解析为 AST**：

1. AST 精确按函数边界切分，归入各模块。
2. AST 精确改写状态引用为 `S.xxx`，跳过：变量声明本身、属性访问（`obj.gameState`）、对象键（`{gameState:}`）、对象简写。
3. 基于符号引用自动生成相对路径 `import`。

## 5. 校验体系（无浏览器环境的唯一保障）

拆分产物在浏览器外验证，运行 `npm run verify`（需 `npm i` 安装 acorn/acorn-walk）：

**`tools/verify-modules.cjs`（静态三查）**
1. 自引用 + link：每个 `import` 的具名符号必须在来源模块真实 `export`。
2. 裸状态引用：36 个状态名不得以裸标识符出现（必须 `S.xxx`）。
3. 缺失 import：用到跨模块导出符号却未 import 的，报错。

**`tools/load-check.mjs`（真实加载）**
用宽容的浏览器全局 stub 让顶层软执行，`import()` 全部 11 个模块，验证 node ESM link 阶段无 "does not provide an export"。这是最接近浏览器的检查。

## 6. 拆分中踩的三个致命坑

1. **ESM 严格模式下顶层重复函数声明是 SyntaxError**：`showSettingsModal` 曾在两处重复定义。经典脚本模式只是后者覆盖前者、不报错，所以 `node --check`（按 CJS 解析）曾漏过。删除残留旧定义后通过。
2. **`acorn-walk` 的 `simple`/`ancestor` 遍历不访问 Pattern 位置的标识符**：赋值左值 `gameState = ...` 处于 Pattern 位置，拆分改写脚本因此漏改 96 处（读取 `S.gameState` 改对了，写入没改）。必须改用 `walk.fullAncestor` 才能覆盖赋值左值。校验脚本一度犯同样的错，产生假阴性——静态验证本身也要验证。
3. **import 清理副作用**：某模块原本只从 `store.js` import 裸状态名，清理后整行删除，导致 `S` 也丢了。需补 `import { S } from "./store.js"`。

> 教训：`node --check` 只做单文件语法解析，**不做跨模块 link**，无法发现"import 了不存在的 export"。必须叠加静态 link 校验 + 真实 ESM 加载。

## 1. 这件事在做什么

把 AetherNarrator（纯前端、零框架、原生 HTML+CSS+JS 的 AI 文字冒险游戏）的界面升级为「PC 与手机都好用」的新设计。设计稿见 `demo.html`，风格基调：**保留暗色羊皮纸 + 金棕 + Noto Serif SC 衬线**，主要改布局与交互，不改品牌。

## 4. 设计规范（新样式必须遵守）

- **断点**：`767px`（≤767 为手机）。与既有 `480px` 查询并存，不冲突。
- **列宽**：阅读/输入统一 `760px` 居中（demo 里写的 720，落地时定为 760，保持一致即可）。
- **颜色**：只用 `:root` / `[data-theme="light"]` 已有变量（`--primary`、`--card`、`--card-border`、`--text`、`--warning` 等），**禁止新增硬编码色值**；金棕半透明遮罩用 `rgba(201,168,124,x)`（深色）/ `rgba(158,125,78,x)`（浅色）。
- **触摸目标**：手机端可点元素 ≥44px 高。
- **安全区**：底部贴边元素加 `env(safe-area-inset-bottom)`。

## 5. 项目铁律（改 UI 绝不破坏功能）

1. **保留 `data-action`**：所有交互靠 `index.html` 的 `data-action` + `src/app.js` 的 ACTIONS 事件委托。改 DOM 不得删除/改错这些属性。
2. **样式集中在 `styles.css`**：不写内联 `style=""`，不用 `!important` 覆盖主题变量。
3. **装饰层 `pointer-events:none`**：`#homeMotes/#gameMotes/.mist/.vignette` 必须保持，否则会挡住点击（真实踩过的坑）。
4. **动态容器只改外壳**：`#gameLog/#worldListContent/#saveListContent/#statusTabs/#statusContent` 等内部由 JS 渲染函数填充，只改外层样式。
5. **弹窗约定**：屏幕 `.screen.active` 显隐；弹窗 `.modal-overlay.show` 显隐；复用 `render.js` 的 `showScreen/showModal/closeModal`。
6. **禁用原生 `confirm()`**：沙箱环境会静默吞掉，确认一律走自定义弹窗。
7. **图谱 canvas**：`#loreGraph` 由 force-graph 渲染，容器尺寸变化需触发既有重算。

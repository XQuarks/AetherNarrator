# 以太叙事 · UI 重设计方案（Demo 版）

> 配套演示：`docs/ui-redesign/demo.html`（浏览器直接打开，顶部控制条可切换 界面 / 设备 / 主题）。
> 本方案为**纯设计稿**，未改动任何正式项目文件（`index.html` / `styles.css` / `src/*` 均未动）。

---

## 1. 设计目标

1. **PC 与手机一套代码兼容**：响应式布局，≥768px 桌面、<768px 移动，同一 DOM 结构只换布局。
2. **保留品牌基因**：暗色羊皮纸 + 金棕 + Noto Serif SC 衬线、星尘/光晕氛围层，老用户一眼认出。
3. **阅读优先**：游戏界面是长文阅读场景，正文列宽收敛到 720px、字号 16px、行高 2.05。
4. **拇指友好（手机端）**：核心操作下沉到底部——底部导航、底部输入区、底部弹层抽屉。

## 2. 关键改动点（对照现有 UI）

| 界面 | 现状问题 | 新方案 |
|---|---|---|
| 首页 | 菜单四条竖排，手机上尚可但无底部导航 | 保留居中仪式感；**手机端新增底部导航**（首页/世界/存档/设置） |
| 世界列表 | 卡片信息密度低、删除按钮与主操作同级易误点 | 卡片化：色块封面 + 两行截断描述 + 标签；主操作「开始/继续」突出，删除降为次级幽灵按钮；PC 双列网格、手机单列 |
| 游戏界面 | 日志通栏铺满，宽屏行长过长难读；输入框单行 | ① 正文列宽 720px 居中；② 玩家行动用右侧气泡区分叙事；③ 输入框改 **textarea 自动增高**（Enter 发送 / Shift+Enter 换行）；④ 选项 chips 固定在输入区上方；⑤ 顶栏浓缩为 56px 毛玻璃 sticky 栏 |
| 角色面板 | 右侧抽屉，手机上挤占阅读区 | 桌面保持右侧抽屉（400px）；**手机改为底部弹层**（高 82%，圆角上沿，下滑关闭），与移动端习惯一致 |
| 知识库 | 三栏在手机上不可用 | 桌面保持 Obsidian 三栏；**手机改为分段切换**（文件/笔记/关联），三栏同一 DOM 切换显隐 |
| 弹窗体系 | 居中弹窗在手机上偏小偏挤 | 统一圆角 18px + 毛玻璃遮罩；手机上弹窗从底部升起（与抽屉同一套动效语言） |
| 主题 | 深/浅双主题已有 | 沿用 CSS 变量方案，新增毛玻璃 `backdrop-filter` 层（降级自动不透明，无兼容风险） |

## 3. 响应式策略

- **断点**：`768px`。Demo 内用 `.device.is-mobile` 类演示，落地时改纯 `@media (max-width:767px)`。
- **安全区**：iPhone 刘海/Home 条用 `env(safe-area-inset-top/bottom)` 处理。
- **触摸目标**：可点元素 ≥36px（icon-btn 36×36，底部导航 58px 高）。
- **meta**：`viewport-fit=cover` 已纳入。

## 4. 落地时的注意事项（沿用参考手册第 5 节铁律）

1. 所有按钮保留 `data-action`，DOM 结构调整不改交互契约。
2. 样式改动集中在 `styles.css` 变量层，不内联硬编码颜色。
3. 装饰层（星尘/雾/暗角）保持 `pointer-events:none`。
4. 动态容器（`#gameLog`/`#worldListContent` 等）只改外层样式，内部渲染函数不动。
5. 弹窗显隐沿用 `.modal-overlay.show` / `showModal` 约定；底部弹层可作为 `.modal-overlay` 的移动端变体（加修饰类）。
6. 图谱 canvas（force-graph）容器尺寸变化时调用既有重算逻辑。

# AI 文字游戏 · 状态与数值设计

本游戏采用**文字描述优先**的状态模型。属性、关系、技能不再用僵硬数字表示，而是用符合世界观的生动描述来呈现。系统只记录 `progression` 等少数进度数值，以及物品数量、目标状态等结构性字段。

## 1. 核心状态模型

```json
{
  "name": "玩家名",
  "age": 16,
  "background": "出身背景",
  "personality": ["谨慎", "好奇"],
  "attributes": {
    "courage": "初来乍到，遇事不免有些畏缩，但还不到仓皇逃窜的地步。",
    "perception": "对周遭动静还算留心，偶尔会注意到旁人忽略的细节。",
    "patience": "能坐得住一时半刻，但若长久无望，也会焦躁起来。",
    "luck": "不好不坏，像被世界随手一扔的普通石子。",
    "will": "心志尚浅，却还没被现实完全磨平。"
  },
  "progression": {
    "path": "路线/学院/分支",
    "rank": "当前境界/年级/等级",
    "progress": 0
  },
  "relationships": {
    "NPC名": "关系的文字描述"
  },
  "skills": {
    "技能名": "技能的文字描述"
  },
  "inventory": [
    { "item_id": "id", "name": "名称", "count": 1 }
  ],
  "completed_events": [],
  "current_location": "地点",
  "current_date": {
    "day": 1,
    "period": "morning"
  },
  "goals": [
    { "goal_id": "id", "name": "目标名", "type": "类型", "deadline": { "day": 3, "period": "night" }, "visible": true }
  ],
  "status_effects": [
    { "name": "状态名", "desc": "描述", "duration": 1 }
  ],
  "is_alive": true,
  "death_reason": null
}
```

## 2. 通用属性（attributes）

五维通用属性，每个键对应一段文字描述：

| 英文键 | 默认中文名 | 含义 |
|--------|-----------|------|
| courage | 胆识/勇气 | 面对危险、冲突的能力 |
| perception | 洞察/观察 | 发现细节、识破伪装 |
| patience | 耐心 | 持久、修炼、等待 |
| luck | 气运/运气 | 随机事件、奇遇 |
| will | 心志/意志 | 抵抗诱惑、坚持目标 |

不同世界观可在 `schema.attribute_labels` 中重新命名。

### 描述示例

**通用奇幻：**

```json
{
  "courage": "才初涉世事，遇事不免踌躇，但骨子里还藏着一股不愿轻易认输的劲。"
}
```

**修仙：**

```json
{
  "courage": "道心初立，虽面对高阶修士仍会紧张，却已敢抬头看对方的眼睛。"
}
```

**哈利波特：**

```json
{
  "perception": "观察力不算敏锐，但偶尔能注意到别人遗漏的魔法细节。",
  "knowledge": "关于英国法律方面，具有实用的知识；植物学知识——不全面，但对于莨蓿制剂和鸦片却知之甚详。"
}
```

> 提示：可以在生成世界时让 AI 根据 IP 与主角设定自定义属性键和描述。例如侦探主题可以新增 `knowledge`、`deduction`；武侠主题可以新增 `qigong`、`poise`。

## 3. 进度系统（progression）

少数保留的数值字段，用于表示阶段性成长：

```json
{
  "path": "修行路线",
  "rank": "境界",
  "progress": 0
}
```

映射示例：

| 世界观 | path | rank | progress |
|--------|------|------|----------|
| 修仙 | 剑修 | 练气期 | 15 |
| 哈利波特 | 格兰芬多 | 一年级 | 0 |
| 蒸汽朋克 | 机械师 | 学徒 | 30 |
| 原创奇幻 | 游侠 | 黑铁 | 5 |

## 4. 关系系统（relationships）

关系用描述而非好感度数值表示：

```json
{
  "老道长": "萍水相逢，他看你的眼神里带着几分打量。",
  "同乡少年": "你们彼此看不顺眼，言语间多有试探。"
}
```

当关系发生质变时，AI 会更新描述。例如：

```json
{
  "老道长": "老道长对你多了几分青眼，话里开始藏着指点。"
}
```

UI 展示时直接显示这段描述，让玩家感受到关系变化，而不是看到一个冷冰冰的 `+1`。

## 5. 技能系统（skills）

通过 `schema.has_skills` 控制是否展示技能页签。

```json
{
  "has_skills": true,
  "skill_label": "功法/技艺"
}
```

技能同样用文字描述：

```json
{
  "剑术": "只会些庄稼把式，连剑都握不太稳。",
  "打坐": "才学会吐纳的皮毛，坐久了腿麻。"
}
```

无技能系统的世界：

```json
{
  "has_skills": false,
  "skill_label": "技能"
}
```

## 6. 物品系统

物品通过 `inventory` 管理：

```json
{ "item_id": "herb", "name": "草药", "count": 1 }
```

UI 中获得的物品会在剧情文本中高亮显示，增强反馈。

## 7. 目标系统

目标分两类：

- **阶段性目标**：在规定时间内完成某个事件或达到某个关系等级。
- **最终目标**：贯穿整个游戏，未达成则失败。

目标状态：

- `pending`：进行中
- `completed`：已完成
- `failed`：已失败

## 8. 时间系统

```json
{
  "current_date": {
    "day": 1,
    "period": "morning"
  }
}
```

默认时段：

- morning（早晨）
- forenoon（上午）
- afternoon（下午）
- evening（傍晚）
- night（夜晚）

每次玩家行动默认推进一个时段，从夜晚推进到早晨时 day + 1。

## 9. 死亡与游戏结束

```json
{
  "is_alive": false,
  "death_reason": "被强敌击杀"
}
```

当 `is_alive` 变为 false 时，弹出游戏结束弹窗，提供返回主界面按钮。

死亡条件可由 AI 根据剧情判定，也可由系统规则判定（如关键属性归零或遭遇致命事件）。

## 10. 状态效果

临时状态效果：

```json
{ "name": "疲惫", "desc": "行动效率降低", "duration": 2 }
```

`duration` 表示持续天数或回合数，可在每天结束时递减。

## 11. AI 更新描述的原则

- **显著变化才更新**：日常闲聊、重复动作不更新描述。
- **质变才更新**：获得新认知、突破瓶颈、关系升温/破裂、技能顿悟、受到重创等。
- **一次不宜更新过多**：通常 1-2 项描述即可。
- **符合世界观**：修仙世界不要说「等级提升」，要说「瓶颈松动，气机流转更顺」；HP 世界不要说「武力提升」，要说「咒语念得更稳了些」。
- **保留旧描述**：如果某字段未变化，AI 应在 `state_changes` 中不包含该键，或设置为空字符串 `""`，系统会保留原描述。

## 12. 兼容性说明

系统同时兼容旧版的数值格式：

```json
{ "courage": 5 }
```

前端会识别类型并做展示降级。但强烈建议新世界统一使用文字描述，以获得更好的叙事体验。

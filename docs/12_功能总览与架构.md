# AetherNarrator 功能总览与架构（沉淀文档）

> 本文档沉淀 C 项目（零后端浏览器端 AI 文字冒险 / 单人跑团引擎）**截至当前已实现的功能**，作为后续开发与交接的索引。
> 更细的单点设计见 `docs/01`~`docs/11`。

---

## 0. 项目定位与运行模型

- **零后端**：纯前端，浏览器运行。API Key 存于 `localStorage`，由玩家自填。
- **多 `<script>` 共享词法全局**：`app-core` / `app-ai` / `app-game` / `map-data` / `combat-*` / `action-menu` / `renderer/tile-map` / `app-ui` 等文件在页面中按序加载，函数与 `var` 声明挂到全局（如 `gameState`、`currentWorld`）。
- **确定性测试体系**：`test/` 下用 Node `vm` 沙箱加载**真实源码**做不耗 API、不依赖 DOM 的单元测试（详见 §12）。

---

## 1. 世界与世界观生成

- 创建入口 `callWorldGenerationLLM` 一次性产出：`schema`（进度/技能/属性中文映射/时间模式）、`initial_state`（主角初始能力、关系、物品、目标、npc_states…）、`lore_kb`（知识库）、`opening_narrative`、`system_prompt`。
- 两种类型：原创世界 / 基于已有 IP（如哈利波特）。支持规则集（dnd 等）、规则自由度（1-5）、世界自由度（1-5）、文风参考。
- 主角能力**不可降级**：初始状态必须如实反映设定，AI 不得擅自降为"初学者"。

---

## 2. 知识库与混合 RAG

- `lore_kb.snippets` 按类别组织：规则 / 地点 / 人物 / 事件 / 物品 / 势力 / 冲突（冲突、事件各≥2 条，含触发条件）。
- **双路检索** + **行为记录滑动窗口**（最近 5 条强制注入），避免长程上下文丢失。
- `computeEmbeddingsForWorld` 预计算向量（与本地模型同格式），检索结果随对话注入。

---

## 3. 状态与数值系统

- **文字描述优先**：属性（胆识/灵慧/耐心/运气/意志）、关系、技能全部用文字描述，仅在显著变化时更新；兼容旧版数值的增量写法。
- **统一时间推进** `applyTimeChange`：消除 `period` 与 `current_date` 双路径相互覆盖；模式 `periods` / `continuous` / `hidden`，时段只能前进不可倒退。
- **目标系统**：主线/支线/个人，带 `deadline`；超时/条件破坏标记 `failed`（不删除，开辟新线）。
- **战斗** `CombatEngine`：D20 掷骰，HP/MP/XP，`spawn_enemies` 触发战斗，`in_combat` 状态贯穿。

---

## 4. 常驻记忆层 `pinned_facts`

- AI 在 JSON 的 `pinned_facts` 标注**一旦成立就永远成立、不可逆**的事实：誓言/承诺、契约/pact、诅咒/祝福、角色死亡（含 NPC）、阵营归属、不可逆世界改变、核心主线目标。
- 被**无条件全量注入每一轮对话**，AI 必须始终遵守、不可遗忘。
- `applyStateChanges` 中有状态信号兜底（死亡 / cursed / oath / pact / allegiance / deaths）自动钉住；玩家**无需手动**添加记忆。
- 详见 `docs/11_常驻记忆层设计.md`。

---

## 5. 预言机 / 混沌因子（oracle）

- **创建时可控**：开关 `oracle.enabled` + 混沌因子 `oracle.chaos_factor`（1-9 滑杆）。
- 关闭 = 「纯爽文」：不制造意外，叙事平滑跟随玩家意图。
- 开启 = 按 `cf/9` 概率在回合中注入"本回合触发随机事件"，对标 Mythic GM Emulator 的 yes/and、no/but 与混沌因子，让单人跑团有"命运的不确定性"。
- 详见 `test/oracle_check.js`（8 项校验）。

---

## 6. 非战斗技能检定

- 接入 TRPG 式 `D20 + 属性调整值` 检定，补全"纯战斗才有骰子"的缺口：
  - `talk` → 说服（CHA，DC13）
  - `use_item` → 巧手（DEX，DC13）
  - `move` → 体操（DEX，DC12）
- `resolveSkillCheck` + `buildActionReport(skill_check)`，结果以剧情内成功/失败反馈，不弹系统窗。
- 详见 `test/skill_check_check.js`（13 项校验）。

---

## 7. 章节 / 幕结构（act）

- `computeCurrentAct` / `updateActProgress`：基于**已完成主线目标数 + 事件数**推导当前"幕"，跨幕时在"剧情"标签页呈现徽章与章节时间线、大事记。
- 跨幕自动把关键进展**钉入 `pinned_facts`**，保证长线叙事不漂移。
- 状态面板含"剧情"标签页（当前幕 + 时间线 + 大事记）。

---

## 8. 地图 ⇄ 战斗 集成（MapDataV1）

- 瓦片地图协议 `MapDataV1`，纯 CSS `div` 渲染（无图片依赖），含 `tile_legend`、`entities`、`poi`、`explored` 等字段。
- **正向链路**：点击地图空地 → BFS 寻路 → `movePlayerOnMap` 移动玩家 → 同步 `current_location` 为瓦片名。
- **战斗 ⇄ 地图**：`addMapEnemiesForCombat` 把战斗敌人显式放到图上（标记 `_combat`）；`syncMapCombatEntities` 每轮对齐（战后清场，保留 AI 叙事放置的敌人）。点地图上的敌人即攻击该敌人。
- 详见 `test/map_combat_check.js`（22 项校验）。

## 9. 反向回写（reverse-writeback）★ 本次新增

- **问题**：§8 正向链路已让"点地图→current_location 同步"，但玩家用自由文本叙事移动时（如"我走向北门"），AI 改了 `current_location`，地图上的玩家标记却停在原地——地图与叙事位置"两张皮"。
- **解决**：`applyStateChanges` 处理完 `map_data` 后调用 `syncMapPlayerFromNarrative(gameState, changes, prevLocation)`，把叙事位置写回地图。
- **定位优先级**：
  1. `changes.player_pos` 显式坐标 `{row, col}`（AI 在叙事移动时给出，**最可靠**）；
  2. `changes.current_location` 命中地图 `poi` 命名节点（名称/子串匹配）；
  3. `changes.current_location` 命中某瓦片图例名（唯一匹配时兜底）。
- **约束**：战斗中位置由战斗逻辑掌控，不回写；`prevLocation` 与新地点相同（未移动）时不回写。
- `placePlayerOnMap` 仅移动 + 揭示迷雾、**不覆盖** `current_location`；`movePlayerOnMap` 在其基础上同步 `current_location`（正向）。

## 10. 战争迷雾（fog of war）★ 本次新增

- 每张地图维护 `explored`（二维布尔网格，**持久化**进存档）与 `fog_of_war` 开关（创建时可选，默认开）。
- **三态渲染**（`TileMap.render`）：
  - 未探索 → 纯黑迷雾格（`tile-fog`，无符号、无实体）；
  - 已探索但当前不可见 → 暗淡（`opacity:0.38`，隐藏动态实体，只留地形记忆）；
  - 当前可见 → 正常渲染地形 + 实体。
- **揭示逻辑**：玩家抵达即 `revealAround` 周围 Chebyshev 半径 1；`computeFogVisible` 在**战斗中额外揭示存活 `_combat` 敌人周围**，保证战场可读。
- **稳健性**：AI 重发同尺寸地图时保留已探索进度（不每次清零）；旧存档无 `explored` 时按玩家起点补初始化。
- 详见 `test/reverse_writeback_fog_check.js`（含迷雾相关 35 项校验中的对应用例）。

## 10.5 势力声望 / 经济 / 合成 / 装备（世界模拟）★ 本次新增

把"世界模拟"从纯叙事推进到可量化的势力博弈与资源循环（对应竞品分析的⑤）：

- **多势力声望 `factions`**：每个势力独立累计 `reputation`（-100~100）与 `stance`（敌视/冷淡/中立/友善/崇敬）。AI 在 `state_changes.factions` 输出 `reputation_delta` + `stance` 驱动；状态面板「势力」标签页渲染各阵营声望条与立场颜色。
- **货币经济 `currency`**：按币种累加 delta（不允许为负），默认 `gold`；世界观可自定义币种（如灵石/银币）。状态面板「收集」页顶部显示持有货币。
- **物品/装备扩展**：`inventory` 的 `add` 支持 `type`/`equippable`/`slot`/`damage_bonus`/`ac_bonus`/`desc`。玩家可在「收集」页一键装备/卸下；**已装备武器提升攻击伤害**（接入 `action-menu.resolveAttack`），已装备盔甲在战斗数值区展示防御加成。
- **合成工坊 `crafting_recipes`**：AI 在叙事中"习得/发现"配方时通过 `state_changes.crafting.add_recipe` 注入（去重）。玩家在「收集」页合成工坊于材料齐全时一键合成——引擎消耗 `inputs` 并产出 `output`，形成"采集 → 习得 → 制作"闭环。
- **动作菜单**：新增「交易」「合成」两个按钮（非战斗、非强制），仅把意图交给 AI，实际经济行为由 AI + `currency`/`inventory` 完成。
- **AI 感知**：`buildCompactGameState` 每轮把 `factions`/`currency`/已知配方（仅 id/name，不泄露 inputs 细节）注入上下文，让 AI 校准阵营与交易行为。
- 向后兼容：旧存档无这些字段时 `applyStateChanges` 自动兜底创建；`defaultInitialState` 写入空结构。
- 详见 `test/faction_economy_check.js`（40 项校验）。

- **数据自洽**：非叙事世界（三国/西游/水浒/星轨遗痕/深窖纪元）内嵌完整 `combat_stats`（6 项 D20 属性 + hp/mp/ac），开打即正常；阵营（`factions`）、货币（`currency`）、装备（`equipped` 引用库存 id）、已知配方（`crafting_recipes`）均预置初始状态。纯叙事世界（山海经/雾港来信）与调查恐怖世界（克苏鲁，`ruleset_type:"cthulhu"`，`has_combat:false`）不设 `combat_stats`，由规则集关闭战斗——克苏鲁以 D100 理智/知识检定推进调查，战斗为最后手段。
- **接入方式**：`js/preset-worlds.js` 定义 8 个工厂函数 + `buildPresetWorlds()`；`index.html` 在 `app-ui.js` 之前加载本文件；`app-ui.js` 的 `loadWorlds()` 首启即从 `buildPresetWorlds()` 注入，并通过迁移逻辑删除旧预设（`demo_红楼梦`/`demo_magic_academy`/`demo_蒸汽与魔法`）后注入新库。旧两个 demo（红楼梦/星辉魔法学院）已按要求移除。
- 详见 `test/preset_worlds_check.js`（302 项校验：结构/lore/属性键一致/规则集/纯叙事无战斗/canon 约束/阵营/装备引用/迁移）。

## 13. 已知边界 / 后续可选项

- C1 安全一致性（主题连点彩蛋）按此前约定**保持原状**，未改动。
- 预设 IP 库（⑪）已落地：8 个开箱即玩世界（5 公有领域 IP：三国/西游/水浒/山海经/克苏鲁 + 3 原创：星轨遗痕/雾港来信/深窖纪元），见 §10.6；旧 2 个 demo（红楼梦/星辉魔法学院）已移除。
- 反向回写的图例名匹配为兜底启发式，精确控制仍依赖 AI 输出 `player_pos`。
- 分支快照（多时间线存档）尚未实现。
- 第⑤项已落地"势力声望/货币/合成/装备"，但**完整商店 NPC（固定货架/定价）尚未做**，交易由 AI 在叙事中即时处理；装备目前仅武器伤害与盔甲信息展示（盔甲 `ac` 未强制改写战斗 `ac` 基准，避免旧存档 AC 丢失）。

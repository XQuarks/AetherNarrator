# 开局剧情从小说原文开头开始 · 改造方案

> 状态：✅ **已落地**（2026-07-20 黎总授权"做"，已改 `src/prompt.js` + 新增 `test/prompt.test.js`，`npm test` 82 项全过）。
> 触发：黎总导入小说原文后，发现游戏开局没从小说最开始走剧情，体验不佳。
> 方向（已 AskUserQuestion 确认）：**AI 从原文开头改写成交互开场**。
> 补充约束（黎总追加）：开场白必须**同时服从读者对世界观的改造**——读者导入哈利波特后若把时间改中世纪 / 主角改赫敏，开局应是"原故事的开端、但整体世界观被改写"，或以赫敏视角展开，而非照搬原著原文开头。

## 一、现状根因（已读磁盘代码确认）

游戏开局展示的开场白来自世界字段 `opening_narrative`（`save.js:73`），它同时被注入 system prompt 当"故事起点"（`prompt.js:259`）。但这个字段的生成链路是：

1. 生成世界时 `game.js:154`：`opening_narrative: generated.opening_narrative || ""`。
2. `generated` 来自 `callWorldGenerationLLM`（`llm.js:36`），其内部一次 LLM 调用由 `buildWorldGenerationPrompt`（`prompt.js:17`）构造 prompt。
3. **问题点**：`buildWorldGenerationPrompt` 的"输出要求"里第 5 项 `opening_narrative`（`prompt.js:116-120`）只写"根据世界观/角色/文风，写出富有氛围感的开场场景"——**完全没要求基于原文开头**，也没要求"从小说第 1 章第 1 段出发"。
4. 同时 `prompt.js:36-38` 的 `sourceSection` 把 `sourceContent` 前 8000 字当"只读参考材料"用于提设定，**没用于开场白**，且未强调"开头"。
5. 结果：LLM 自由发挥了一个不相关的引入式开头（甚至真实路径常返回空 → 落到 `save.js:75` 通用模板"你进入了「世界名」…旅程即将开始"）。

**结论**：小说原文被当"设定素材"读了，开场白与原文开头脱钩 → 黎总感觉"没从最开始"。

## 二、目标

导入小说原文后，开局剧情**从小说第 1 章第 1 段出发**，且**严格服从读者对世界观的改造**：

- AI 读取原文最开始部分，改写成第二人称"你…"的互动式开场白，保留核心场景/出场人物/氛围；
- **若读者把主角改成了其他人**（如哈利→赫敏）：开局以**该角色的视角/身份**展开（"你，赫敏…"），而非原著主角视角——即"不是哈利波特原文开头，而是赫敏视角的游戏开始"；
- **若读者改了世界观**（如时间→中世纪、地点/力量体系调整）：场景、服饰、器物、社会规则都按**调整后的设定**重写，呈现"原故事的开端、但整体世界观已被改写"的体验；
- 两者可同时生效（主角也改、世界观也改）。

> 数据现状利好：读者改造分别存在 `world.desc`(世界观描述) 与 `world.hero`(主角设定) 中，生成世界时**已经**传入 `buildWorldGenerationPrompt`（`prompt.js:52-53`）；原文开头也已在 prompt 的 `<reference_material>`（`prompt.js:36-38`）。本次**只需把开场白这条要求改"显式引用这些已有字段"**，不新接任何数据管道。

## 三、方案（精准最小改动，零回归）

只改**生成世界的 prompt 构造**，不动运行时/召回/剧情引擎（startGame、system prompt 注入"故事起点"逻辑已正确消费 `opening_narrative`，只要内容对了就自动从最开始）。

### 改动 1：`src/prompt.js` — `buildWorldGenerationPrompt`
- 新增常量 `OPENING_SRC_CHARS = 3000`（默认取原文前 3000 字，约第一章开头；可调）。
- 新增注入段【原文开头】（当 `sourceContent` 存在）：把 `sourceContent.slice(0, OPENING_SRC_CHARS)` 包进 `<opening_source>`，明确标注"这是源文件最开始部分 / 故事开篇，请据此生成开场白"。放在 `sourceSection` 之后（与既有 `<reference_material>` 互补：前者聚焦"开头起点"、后者供抽设定）。
- 改写"输出要求"第 5 项 `opening_narrative`（`prompt.js:116-120`）为：
  ```
  5. opening_narrative: 开场白字符串（1-3段），玩家进入故事的第一句话，必须从故事开端出发。要求：
     - 【起点】若上方有【原文开头】段：必须以其为叙事起点，从原文第 1 章第 1 段情境出发（保留核心场景、出场人物、氛围），改写为第二人称"你…"的互动式开场。
     - 【服从读者改造·关键】开场白必须严格服从读者调整后的【世界观描述】（上方已给）与【主角设定 hero】（上方已给），而非照搬原著原文样貌：
         · 若【主角设定】把主角改成其他人（如哈利→赫敏）：开局以该角色视角/身份展开（"你，赫敏…"），而非原著主角视角；
         · 若【世界观描述】调整了时间/地点/力量体系/社会结构（如改中世纪）：场景/服饰/器物/社会规则都按调整后设定重写，呈现"原故事开端、但整体世界观已被改写"；
         · 两者都改则同时生效。
     - 让玩家立即感受身处该世界、知晓处境与初步目标
     - 结尾暗示第一个行动方向，但不要强制
     - 篇幅适中（200-500字）
     - 兜底：无【原文开头】段（纯描述生成的世界）时，按原逻辑基于【世界观描述】+【主角设定】写富有氛围感的开场。
  ```
  - 说明：生成世界的 prompt 已含 `desc`/`hero` 与原始 `<reference_material>`，此处只是让开场白这条要求**显式引用**它们，无需新增数据字段。

### 改动 2（可选，非必须）：`src/llm.js` — `mockGenerateWorld` 兜底模板
- 模拟模式本就不读真实原文，保持现三选一模板即可（不影响真实 API 用户）。方案默认**不动** mock，仅在文档标注。

- `rag.js` / `game.js` 召回与剧情引擎。

## 六、范围确认点（默认推荐，可调整）
1. **原文开头截取长度**：默认 3000 字（约第一章开头）。可调到 1500 / 整本前 1.5 万（Plan A 首段）。
2. **适用条件**：仅当导入了 `sourceContent`（小说原文）时生效；纯描述生成的世界保持现状（AI 基于 desc 写开场）。
3. **mock 模拟模式**：保持模板（不读真实原文），不改。
4. **服从读者改造的粒度（本次核心新增）**：默认**同时**服从 `world.desc`(世界观) 与 `world.hero`(主角)；主角被改→开场切该角色视角，世界观被改→场景按新设定重写。
   - 另注（非本次范围，留作未来可选）：若读者**生成世界之后**才改 desc/hero，本次开局白不会自动重算（它定格在生成时）。如需"改完世界观一键重生成开场"，是独立小功能，可后续单立——本次不做，避免扩大回归面。

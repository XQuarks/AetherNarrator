## 一、项目自身审查（家底盘点）

### 1.1 定位与架构

| 维度 | 现状 |
|---|---|
| 产品定位 | 纯前端（零后端）AI 文字游戏/叙事模拟器，玩家可自定义 IP 世界观、创建角色、用 AI 生成剧情，支持本地知识库(RAG)与存档。类 AI Dungeon / 猫箱 / SillyTavern。 |
| 部署形态 | 原生 ESM、零构建，`index.html` 仅加载 `src/app.js`，可直接 GitHub Pages / 静态服务器运行；全部数据存浏览器本地（IndexedDB + 少量 localStorage）。 |

| AI 接入 | 直连模型服务商 OpenAI 兼容 API（默认 DeepSeek `deepseek-v4-flash`），支持 SSE 流式、断流降级、CORS 代理、Mock 体验模式。 |
| 文档资产 | `docs/` 下 20 篇设计/测试文档，质量较高（RAG 方案、Prompt 工程、状态数值、行为记录、安全核查等）。 |

### 1.4 工程亮点（对外可主打的卖点）

1. **会"自己往前走"的世界**：事件引擎 + 目标链 + NPC 自主活动，多数竞品（含猫箱/星野/筑梦岛）本质是**单线对话**，这是最清晰的差异化。
2. **纯前端 + 本地隐私**：数据不出本机，无账号、无上云；对隐私敏感用户是强卖点（猫箱也主打端到端加密，但数据仍在字节云）。
3. **中文 RAG + 世界观守卫**：原生中文分词、IP 违禁概念过滤，贴合中文同人/IP 模拟。
4. **文字描述优先的数值体验**：避免数字类型污染，体验更"叙事"。

### 1.5 当前短板 / 技术债

1. **无后端（根本约束）**：CORS 痛点、API Key 直连泄露面、无法做云同步/多用户/协作/社区/商业化/内容审核。
2. **向量能力受限**：浏览器 CPU 推理、无 GPU；**已有 ANN 近似向量索引（按 worldId 内存缓存，检索非线性扫描）**，但索引未持久化到 IndexedDB（每次刷新页面重建一次）；首次为全量 lore 算 embedding 仍有卡顿（已用 Worker + 构建期预计算默认世界向量缓解，且中文已换 bge-small-zh-v1.5）。
3. **单模型锁定**：仅 DeepSeek 优化。
4. **无移动端专门适配**：桌面优先布局，未响应式/触控优化。
5. **无多模态**：无 TTS 语音、无 AI 生图、无角色立绘。
6. **无社区/UGC 分享**：纯单机，记忆包仅本地 JSON 导入导出。
7. **记忆无可视化时间线 / 跨会话长期记忆**（mem0 式）。
8. **Lorebook 插入位仅 1 处**（user 末尾），竞品可选 5 处。
9. **测试覆盖中等**：`test/` 有 9 个 Node 单测，但无浏览器级 E2E/CI。
10. **代码债（已部分清理）**：根目录死文件已删除（双份源码并存问题已解决）；`game.js` 已拆出 `save.js`/`lore-ui.js`；剩余：`S` 容器读写分散、`utils.js` 与 `game.js` 历史上曾循环 import（已用 `import` 清理缓解）。

## 二、竞品调研（精选最佳）

### 2.1 竞品选型逻辑

| 类别 | 代表 | 入选理由 |
|---|---|---|
| 海外标杆（功能成熟） | **AI Dungeon**、**Character.AI** | 赛道开创者，记忆系统/角色生态最成熟 |
| 开源王者（技术最透明） | **SillyTavern / TavernAI** | 28k+ star，架构开源，Lorebook/扩展/多模型全有 |
| 中文产品（最接近你的用户群） | **猫箱(字节)**、**筑梦岛(网易)**、**星野(MiniMax)** | 国内主流，UGC/社区/多模态/商业化可直接对标 |
| 开源新锐（技术范式前沿） | **Kagari**、**Narratium.ai**、**aiMUD**、**r.A.I.d**、**MemoryForge**、**仙途** | 小模型友好、Graph-RAG、记忆核心、知识图谱等前沿思路 |

### 2.2 各竞品功能与技术点

#### A. AI Dungeon（Latent Space）
- **功能**：无限自由文字冒险；Memory Bank（AI 自动摘要每 4~6 个动作为一条记忆并 embedding 入库，超限时按相关度召回注入）；Story Cards（类似 Lorebook）；Author's Note（中部注入位）；Plot Essentials；Context Viewer（**时间线/相关度双视图**看记忆占用）；自动故事摘要。
- **技术**：embedding + 向量检索；上下文分区预算（Memory Bank / Author's Note / Story Cards 各占配额）；记忆"遗忘"机制（最久未用淘汰）；源自 Voyage 的记忆系统。
- **可学**：① 记忆的"时间线视图"可视化；② 上下文配额分区；③ 自动摘要 + 遗忘淘汰。

#### B. SillyTavern / TavernAI（开源，AGPL-3.0，28k+ star）
- **功能**：**多后端路由**（OpenAI/Claude/Mistral/DeepSeek/KoboldCpp/Ollama/AI Horde 等 20+）；**角色卡 Character Cards**（JSON/PNG，v2 含 Alt Greetings）；**WorldInfo / Lorebooks**（关键词触发的 RAG）；**Group Chats 多角色共演**；视觉小说模式；内置 RAG（文档注入）；**扩展系统**（角色表情/自动摘要/向量存储/Stable Diffusion 生图/TTS/联网搜索）；采样参数全暴露；移动端响应式。
- **技术**：**Node.js(Express) 后端**做 API 代理/密钥/TTS/生图；原生 JS + jQuery 前端；扩展插件机制；支持 Chat/Text Completion 双结构。
- **可学（最重要）**：① 多模型抽象层（统一 OpenAI 兼容接口）；② Lorebook **多插入位**（system/author note/before user/after user 等 5 处）；③ 角色卡 v2 标准兼容（互导获客）；④ 扩展/插件架构；⑤ 群聊多角色。

#### C. 猫箱（字节跳动，豆包大模型）
- **功能**：沉浸式剧情互动 + AI 虚拟角色陪伴；**自定义角色**（外貌/音色/性格/背景，38 种模板）；**沉浸剧情**（可化身角色语C/剧本杀，故事线编辑器 + 视觉化生成器）；**多角色共演**；多模态（文字/语音/图像生成）；**UGC 社区 + 角色交易所**（热门角色下载超 500 万次，形成虚拟经济）；会员体系 + "猫粮"虚拟币付费；端到端加密隐私；语音延迟 0.8s、文字 300 字/s。
- **技术**：豆包大模型（130 亿参数，中文语义理解 +27%）；火山引擎 AI 服务矩阵；模型轻量化；端到端加密。
- **可学**：① UGC 社区/角色市场（获客与留存核心）；② 故事线可视化编辑器；③ 多模态（语音+生图）；④ 付费/虚拟经济模型。

#### D. 筑梦岛（网易，背靠阅文）
- **功能**：1对1 单聊 + **群聊（伪群聊）**；**小剧场**（人设+场景的对话模式）；**朋友圈/动态**（AI 发动态、有情绪变化，"像真人社交"）；云端梦境保存；穿越小剧场；语音通话；免费版约 50 条/天限制 + 会员。
- **技术**：内容 IP 资源（阅文）；角色背后是高 Quality Prompt + 工程/模型优化。
- **可学**：① **主动互动/朋友圈动态**（"你不找它，它会找你"——解决 AI 伴侣留存关键）；② 小剧场（人设×场景）；③ 群聊。

#### E. 星野（MiniMax）
- **功能**：2D 动漫立绘精致；角色创建（外貌/性格/背景/说话风格）；文字 + **语音**（Speech 系列拟人化声音，5 秒克隆）；**CG 卡牌抽卡**收集机制；免费为主。
- **技术**：MiniMax 语音模型（情绪/语气/笑声、超长文本合成）；文生图立绘。
- **可学**：① 多模态语音沉浸；② **抽卡/收集**游戏化机制（提升活跃与付费）。

#### F. Character.AI（全球最大）
- **功能**：百万级用户角色；角色人设维持最强（聊 50 轮不忘设定）；群聊；但**对话结束即失忆、无关系成长、纯文字、无主动互动**；英文为主、需科学上网、Plus $9.99/月。
- **可学（反面）**：它验证了"角色扮演"天花板——**纯聊天无记忆成长、无主动互动会流失**；提醒我们"世界自主演进 + 记忆成长"路线的正确性。

#### G. 开源新锐（技术范式前沿）
| 项目 | 功能亮点 | 技术亮点 | 可学 |
|---|---|---|---|
| **Kagari** | 体验驱动、小模型友好的 AI 叙事框架 | React + PWA；路线图中"记忆核心=RAG+知识图谱"把记忆从上下文搬到库；动态 UI Agent（LLM 生成专属 UI） | ① 知识图谱+向量混合；② PWA 移动端；③ 小模型友好降本 |

## 四、我方缺失 / 落后的功能（按维度汇总）

1. **部署形态**：无后端 → CORS、无云同步、无多用户、无协作、无内容审核、无商业化计费。
2. **模型层**：仅 DeepSeek 优化，无多模型抽象；切换模型 token 暴涨、缓存失效。
3. **知识库高级特性**：Lorebook 仅 1 插入位（竞品 5 选）；向量索引未持久化到 IndexedDB（每次刷新页面重建一次）；首次为全量 lore 算 embedding 仍有卡顿（已用 Worker + 构建期预计算缓解，中文已换 bge-small-zh-v1.5）。
4. **记忆系统**：无可视化时间线/回溯/分支；无跨会话长期记忆；无分层(短/长)与知识图谱混合。
5. **互动形态**：无群聊/多角色共演 UI；无主动互动/朋友圈动态（筑梦岛式"AI 找你"）；无多模态（语音/生图/立绘）。
6. **社区与增长**：无 UGC 社区/角色市场；角色卡不与 SillyTavern v2 标准互通（错失现成生态流量）。
7. **移动端**：桌面优先，无响应式/App/PWA。
8. **商业化与合规**：无付费体系；若做 UGC 则缺内容审核与 IP 合规护栏。
9. **工程化**：无 E2E/CI；`game.js` 过大；根目录死文件待清；循环 import 风险。

## 六、总结与建议路线图

**我方当前位置**：一个**功能完整、工程质量中上、且有清晰差异化（世界自主演进 + 本地隐私 + 中文 RAG）**的纯前端 AI 叙事引擎。与三大成熟竞品相比，核心玩法已不缺，短板已从"功能缺失"转为"部署形态（无后端/单用户/单模型/移动端）"与"少数高级记忆特性（时间线/跨会话）"。

**路线建议（按你"先登记、确认再做"的节奏）**：

1. **守成（现在就有，别动）**：事件引擎、本地隐私、中文 RAG、世界观守卫——这是护城河。
2. **近期 P0（低风险高回报）**：多模型抽象 → 多插入位 Lorebook → 中文向量索引 → 清理死文件。这四项纯前端可落地，不动架构。
3. **中期 P1（拉开差距）**：记忆时间线可视化、Graph-RAG、主动互动、群聊、角色卡互通。
4. **远期 P2（看是否商业化）**：多模态、轻后端+云、UGC 社区、移动端、合规审核。

> 一句话：**先吃下 SillyTavern 的"多模型 + 多插入位 + 角色卡互通"三件套（P0），你就能在纯前端形态下立刻追平 80% 的竞品体验；再用"世界自主演进"这个别人没有的差异化去打差异。** 社区/多模态/后端是"做不做大"的选择，不是"能不能玩"的必需。

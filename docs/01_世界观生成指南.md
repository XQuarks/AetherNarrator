# AI 文字游戏 · 世界观生成指南

本指南用于辅助 AI（或开发者）为「AI 文字游戏模拟器」生成一个可玩的、结构化的世界观。

## 1. 世界观的输入

创建世界时，用户通常提供以下信息：

- **世界名称**：如「剑来」「哈利波特」「原创蒸汽朋克」。
- **类型**：
  - `ip`：基于已有小说 / 动漫 / 影视 / 游戏。
  - `original`：完全原创。
- **世界观描述**：力量体系、社会结构、主要势力、关键地点、核心规则、时代背景。
- **主角设定**（可选）：出身、性格、初始状态、目标。

## 2. AI 要生成的输出

基于输入，AI 应输出一个完整的 JSON 配置，包含：

### 2.1 schema（世界属性模板）

定义该世界在 UI 中如何展示角色属性：

```json
{
  "progression_label": "境界",
  "progression_path_label": "修行路线",
  "has_skills": true,
  "skill_label": "功法/技艺",
  "attribute_labels": {
    "courage": "胆识",
    "perception": "洞察",
    "patience": "耐心",
    "luck": "气运",
    "will": "心志"
  },
  "time_periods": {
    "morning": "早晨",
    "forenoon": "上午",
    "afternoon": "下午",
    "evening": "傍晚",
    "night": "夜晚"
  },
  "game_over_conditions": ["is_alive === false"]
}
```

**设计原则**：

- `progression_label` 要贴合世界观：修仙用「境界」，魔法学院用「年级」，蒸汽朋克用「职业等级」。
- `has_skills` 控制是否显示技能页签。没有技能系统的世界设为 false。
- `attribute_labels` 提供通用的五维属性中文名，可按世界观调整。
- `time_periods` 定义一天内的时段，可增删。

### 2.2 initial_state（玩家初始状态）

```json
{
  "name": "少年",
  "age": 16,
  "background": "小镇出身，机缘巧合踏入修行路。",
  "personality": ["谨慎", "坚韧"],
  "attributes": {
    "courage": "才初涉世事，遇事不免踌躇，但骨子里还藏着一股不愿轻易认输的劲。",
    "perception": "对周遭动静还算留心，偶尔会注意到旁人忽略的细节。",
    "patience": "能坐得住一时半刻，但若长久无望，也会焦躁起来。",
    "luck": "不好不坏，像被世界随手一扔的普通石子。",
    "will": "心志尚浅，却还没被现实完全磨平。"
  },
  "progression": { "path": "未入门", "rank": "凡人", "progress": 0 },
  "relationships": {
    "老道长": "萍水相逢，他看你的眼神里带着几分打量。",
    "同乡少年": "你们彼此看不顺眼，言语间多有试探。"
  },
  "skills": {
    "剑术": "只会些庄稼把式，连剑都握不太稳。",
    "打坐": "才学会吐纳的皮毛，坐久了腿麻。"
  },
  "inventory": [{ "item_id": "bread", "name": "干粮", "count": 2 }],
  "completed_events": [],
  "current_location": "小镇入口",
  "current_date": { "day": 1, "period": "morning" },
  "goals": [
    { "goal_id": "find_shelter", "name": "找到落脚之处", "type": "完成事件", "deadline": { "day": 1, "period": "night" }, "visible": true }
  ],
  "status_effects": [],
  "is_alive": true,
  "death_reason": null
}
```

**设计原则**：

- `attributes` / `relationships` / `skills` 全部使用**文字描述**，不要数字。描述要体现当前水准与世界观。
- `progression` 用通用字段 path/rank/progress，避免用 cultivation 等修仙专属词。
- `goals` 必须包含阶段性小目标和最终目标，可设置 deadline。
- `is_alive` 必须存在，用于游戏结束判定。

## 3. 世界观一致性检查清单

生成后，检查：

- [ ] 力量体系是否自洽？（境界/年级/职业等级不能跳跃）
- [ ] 是否有不可篡改的核心剧情或角色命运？
- [ ] NPC 是否有初始态度和帮助门槛？
- [ ] 死亡条件是否合理？
- [ ] 物品、技能、地点是否属于该世界？
- [ ] 初始目标是否明确且可达成？

## 4. 不同类型世界的注意事项

### 4.1 修仙 / 武侠

- 境界必须有序列，如「凡人→练气→筑基→金丹」。
- 重大突破需要特殊事件，不能一次行动直接突破。
- 功法、法宝、丹药要有获取门槛。

### 4.2 魔法学院（哈利波特型）

- 用「年级」替代「境界」，用「学院」替代「修行路线」。
- 低年级学生不能施展高级咒语。
- 课程/法术作为技能系统。

### 4.3 蒸汽朋克 / 科幻

- 用「职业等级」或「技术等级」。
- 装备、机械、资源管理是核心。
- 可关闭技能系统，用「技术树」替代。

### 4.4 原创世界

- 先定义核心规则和三到五个关键词。
- 避免与知名 IP 设定冲突。
- 所有内容要能在 System Prompt 中讲清楚。

## 5. 输出格式要求

AI 生成世界观时，必须输出合法 JSON，不要包含 markdown 代码块。字段名和结构必须严格遵循本指南，否则前端无法正确解析。
